#+TITLE: org-tools — Release Process

* Pre-release Checklist

1. All tests pass: =cargo test=
   - [ ] Verify: exit code 0, no failures
2. Clippy clean: =cargo clippy -- -D warnings=
   - [ ] Verify: no warnings
3. Edge case fixtures pass: =cargo run -- check tests/fixtures/edge_cases/=
   - [ ] Verify: no panics, expected diagnostics only
4. Version bump in =Cargo.toml=
   - [ ] Verify: version field updated
5. Update changelog (generated from conventional commits)
   - [ ] Verify: changelog reflects all changes since last release

* Release Steps

1. Create a release commit:
   #+begin_src sh
   git add Cargo.toml Cargo.lock
   git commit -m "chore: release vX.Y.Z"
   #+end_src

2. Tag the release:
   #+begin_src sh
   git tag -a vX.Y.Z -m "Release vX.Y.Z"
   #+end_src

3. Push:
   #+begin_src sh
   git push origin main --tags
   #+end_src

4. Publish to crates.io:
   #+begin_src sh
   cargo publish
   #+end_src
   - [ ] Verify: =cargo install org-tools= installs the new version

* Post-release

- [ ] Verify CI passed on the tag
- [ ] Verify crates.io page shows correct version and description
