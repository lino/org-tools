#+TITLE: Org-Mode Ecosystem: Packages That Extend Syntax
#+AUTHOR: org-tools

* Overview

This document catalogs org-mode adjacent packages and the syntax they
add. A standalone linter/formatter needs to understand these extensions
to avoid false positives on valid files.

Sources:
- [[https://orgmode.org/manual/][Org Mode Manual]]
- [[https://orgmode.org/worg/org-syntax.html][Org Syntax (formal spec)]]
- [[https://github.com/200ok-ch/org-parser/blob/master/resources/org.ebnf][200ok org-parser EBNF grammar]]
- [[https://git.savannah.gnu.org/cgit/emacs/org-mode.git/tree/lisp/org-lint.el][org-lint.el source]]

* Packages That Extend Org Syntax

** Built-in Extensions (part of org-mode)

*** org-id — Unique identifiers
- Spec: [[https://orgmode.org/manual/Handling-Links.html][§4 Handling Links]]
- Property: ~:ID:~ — globally unique identifier (UUID)
- Link type: ~id:UUID~ — link to heading by ID

*** org-attach — File attachments
- Spec: [[https://orgmode.org/manual/Attachments.html][§10.2 Attachments]]
- Properties: ~:DIR:~, ~:ATTACH_DIR_INHERIT:~
- Link type: ~attachment:filename~

*** org-crypt — Entry encryption
- Spec: [[https://orgmode.org/manual/Org-Crypt.html][§21.1 Org-Crypt]]
- Tag: ~:crypt:~ — marks heading for encryption
- Property: ~:CRYPTKEY:~ — GPG key ID

*** org-habit — Habit tracking
- Spec: [[https://orgmode.org/manual/Tracking-your-habits.html][§5.3.3 Tracking your habits]]
- Property: ~:STYLE: habit~
- Property: ~:LAST_REPEAT:~ — auto-managed timestamp
- Scheduling: ~.+Nd/Md~ repeater syntax for min/max intervals

** External Packages — High Popularity

*** org-roam — Zettelkasten/wiki (6k+ stars)
- Repo: [[https://github.com/org-roam/org-roam]]
- Properties:
  - ~:ROAM_ALIASES:~ — alternative node names
  - ~:ROAM_REFS:~ — reference identifiers (URLs, cite keys)
  - ~:ROAM_EXCLUDE:~ — exclude from database (value: ~t~)
- Relies on ~:ID:~ from org-id

*** org-ref — Citations & cross-references (1.4k+ stars)
- Repo: [[https://github.com/jkitchin/org-ref]]
- Adds ~50+ citation link types~ (natbib + biblatex):
  - Natbib: ~cite~, ~citet~, ~citep~, ~citealt~, ~citealp~,
    ~citenum~, ~citeauthor~, ~citeyear~, ~citeyearpar~
    (plus starred and capitalized variants)
  - Biblatex: ~Cite~, ~parencite~, ~footcite~, ~textcite~,
    ~smartcite~, ~autocite~, ~fullcite~, ~supercite~
    (plus starred, capitalized, and multi variants)
  - Cross-ref: ~ref~, ~eqref~, ~pageref~, ~nameref~, ~autoref~, ~label~
  - Bib: ~bibliography~, ~bibliographystyle~
- All citation links use ~&~ as key prefix: ~cite:&key~

*** ox-hugo — Hugo export (1.9k+ stars)
- Repo: [[https://github.com/kaushalmodi/ox-hugo]]
- Keywords (~#+HUGO_*~):
  - ~#+hugo_base_dir:~, ~#+hugo_section:~, ~#+hugo_tags:~,
    ~#+hugo_categories:~, ~#+hugo_draft:~, ~#+hugo_weight:~,
    ~#+hugo_publishdate:~, ~#+hugo_expirydate:~,
    ~#+hugo_auto_set_lastmod:~, ~#+hugo_custom_front_matter:~,
    ~#+hugo_front_matter_key_replace:~, ~#+hugo_aliases:~,
    ~#+hugo_menu:~, ~#+hugo_outputs:~, ~#+hugo_resources:~,
    ~#+hugo_bundle:~
- Properties (~:EXPORT_HUGO_*~):
  - ~:EXPORT_FILE_NAME:~, ~:EXPORT_HUGO_SECTION:~,
    ~:EXPORT_HUGO_BUNDLE:~, ~:EXPORT_HUGO_DRAFT:~,
    ~:EXPORT_HUGO_WEIGHT:~, ~:EXPORT_HUGO_TAGS:~,
    ~:EXPORT_HUGO_CATEGORIES:~, ~:EXPORT_HUGO_CUSTOM_FRONT_MATTER:~

*** org-reveal — Reveal.js presentations (2.6k+ stars)
- Repo: [[https://github.com/yjwen/org-reveal]]
- Keywords (~#+REVEAL_*~):

- Properties: ~:reveal_background:~, ~:reveal_background_trans:~

*** org-noter — Document annotation (900+ stars)
- Repo: [[https://github.com/weirdNox/org-noter]]
- Properties:
  - ~:NOTER_DOCUMENT:~ — path to annotated PDF/EPUB
  - ~:NOTER_PAGE:~ — page/location of annotation

*** denote — Simple note-taking (1.2k+ stars)
- Repo: [[https://github.com/protesilaos/denote]]
- Keyword: ~#+identifier:~ — unique note ID (timestamp-based)

*** org-journal — Journaling (800+ stars)
- Repo: [[https://github.com/bastibe/org-journal]]

** External Packages — Medium Popularity

*** org-edna — Task dependencies (GNU ELPA)
- Repo: [[https://savannah.nongnu.org/projects/org-edna-el/]]
- Properties with mini-language:
  - ~:BLOCKER:~ — conditions before task can complete
  - ~:TRIGGER:~ — actions when task is marked DONE
- Mini-language finders: ~ids(...)~, ~self~, ~previous-sibling~,
  ~next-sibling~, ~children~, ~parent~, ~match(...)~, ~file(...)~
- Mini-language actions: ~todo!(STATE)~, ~scheduled!(TIMESPEC)~,
  ~deadline!(TIMESPEC)~, ~set-property!(PROP VAL)~, ~tag!(TAG)~
- org-tools support:
  - W035 (~invalid-edna-syntax~) validates BLOCKER/TRIGGER property syntax
  - ~blocked~ and ~actionable~ query predicates evaluate edna BLOCKER
    conditions to determine whether an entry is ready to work on

*** org-drill — Flashcards/spaced repetition (org-contrib)
- Properties (~:DRILL_*~):
  - ~:DRILL_CARD_TYPE:~ — twosided, multisided, hide1cloze, etc.
  - ~:DRILL_LAST_INTERVAL:~, ~:DRILL_REPEATS_SINCE_FAIL:~,
    ~:DRILL_TOTAL_REPEATS:~, ~:DRILL_FAILURE_COUNT:~,
    ~:DRILL_AVERAGE_QUALITY:~, ~:DRILL_EASE:~,
    ~:DRILL_LAST_QUALITY:~, ~:DRILL_LAST_REVIEWED:~
- Inline cloze syntax: ~[hidden text]~, ~[hidden text||hint]~
- Tag: ~drill~

*** org-fc — Modern flashcards (600+ stars)
- Repo: [[https://github.com/l3kn/org-fc]]
- Properties: ~:FC_TYPE:~, ~:FC_CREATED:~, ~:FC_CLOZE_MAX:~, ~:FC_CLOZE_TYPE:~
- Drawer: ~:REVIEW_DATA:~ — stores scheduling data

*** org-brain — Concept mapping (1k+ stars)

*** org-transclusion — Content transclusion (700+ stars, GNU ELPA)

*** anki-editor — Anki sync (600+ stars)
- Repo: [[https://github.com/louietan/anki-editor]]

  ~:ANKI_TAGS:~

*** org-gtd — GTD workflow (500+ stars)

  ~:ORG_GTD_PROJECT_IDS:~, ~:ORG_GTD_DEPENDS_ON:~

*** ox-pandoc — Pandoc export (300+ stars)
- Repo: [[https://github.com/kawabata/ox-pandoc]]

  ~#+PANDOC_VARIABLES:~, ~#+PANDOC_EXTENSIONS:~

*** toc-org — Auto table of contents (600+ stars)

* Differences: 200ok org-parser vs Official Spec

Ref: [[https://github.com/200ok-ch/org-parser/blob/master/resources/org.ebnf][org-parser EBNF grammar]]

** Unsupported in org-parser
- Nested text markup (~*bold /italic/ text*~)
- LaTeX environments (~\begin{env}...\end{env}~)
- Most LaTeX fragments (~\(...\)~, ~\[...\]~, ~$$...$$~, ~$...$~)
- Export snippets (~@@backend:value@@~)
- Inline babel calls (~call_name(args)~)
- Inline source blocks (~src_lang{body}~)
- Statistics cookies (~[50%]~, ~[3/5]~)
- Citations (~[cite:@key]~)
- Inlinetasks (15+ star headings)
- Radio links
- Numeric priorities (~[#0]~ through ~[#64]~)

** Partial/different in org-parser
- Lists parsed line-by-line only (no nesting/termination semantics)
- Block content matched greedily (may not stop at first ~#+END_~)
- Block name not validated between begin/end
- Drawer nesting allowed (spec forbids it)
- Anonymous footnotes cannot contain brackets
- Src block switches/header args not parsed
- Link descriptions not parsed for org objects
- Timestamps accept AM/PM and seconds (beyond spec)
- TODO keywords restricted to uppercase ASCII only

* Implications for org-tools

** What org-tools should NOT flag as errors
1. Any ~#+KEYWORD:~ starting with a known package prefix (~HUGO_~,
   ~REVEAL_~, ~PANDOC_~, etc.) — treat as valid
2. Properties starting with known package prefixes (~ROAM_~, ~DRILL_~,
   ~FC_~, ~BRAIN_~, ~ANKI_~, ~NOTER_~, ~ORG_GTD~, etc.)
3. Custom block types (~#+BEGIN_NOTES~, ~#+BEGIN_RESULTS~, etc.)
4. Custom link types (~cite:~, ~attachment:~, ~id:~, etc.)
5. Package-specific tags (~:crypt:~, ~:drill:~, ~:fc:~, ~:TOC:~)

** What org-tools should consider for keyword-validity rule
The ~keyword-validity~ lint rule should maintain an allowlist of known
keyword prefixes that covers both core org-mode and popular packages:
- Core: ~TITLE~, ~AUTHOR~, ~DATE~, ~OPTIONS~, ~STARTUP~, etc.
- ox-hugo: ~HUGO_*~
- org-reveal: ~REVEAL_*~
- ox-pandoc: ~PANDOC_*~
- org-transclusion: ~TRANSCLUDE~
- denote: ~IDENTIFIER~
- General: ~EXPORT_*~, ~ATTR_*~, ~LATEX_*~, ~HTML_*~

Properties are user-defined by nature, so org-tools should never flag
unknown property keys — only validate syntax, not semantics.
