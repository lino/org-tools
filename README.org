#+TITLE: org-tools
#+OPTIONS: toc:2

Unified org-mode CLI: lint, format, query, clock, export. Runs without Emacs.

* Why

For almost ten years I maintained a janky toolchain of Python scripts and
Emacs Lisp hacks to keep my org files consistent. It worked — mostly — until
it didn't: scripts broke on edge cases, elisp ran only inside Emacs, and
nothing could run in CI without pulling in a full Emacs installation.

org-tools replaces all of that with a single Rust binary. It is fast, correct
(rules are validated against the org-mode specification and Emacs source),
and designed to run anywhere — locally, in CI, or from any editor.

This project was built with the help of AI tooling (Claude Code). The rule
set was designed by studying [[https://git.savannah.gnu.org/cgit/emacs/org-mode.git/tree/lisp/org-lint.el][org-lint.el]] and the [[https://orgmode.org/worg/org-syntax.html][org-mode specification]].

* Features

** Lint & Format (=org fmt=)

- *9 format rules* (auto-fixable): trailing whitespace, blank lines, heading
  spacing, table alignment, property drawer alignment, keyword spacing, tag
  alignment, list formatting, timestamp spacing
- *37 lint rules* (report-only): structural validation, correctness checks,
  deprecation warnings, style suggestions — covering ~35 of Emacs org-lint's
  ~58 checks
- *Protected regions*: source, example, export, quote, verse, center, and
  comment blocks are never modified by format rules
- *Configurable* via =.org-tools.toml= with sensible defaults matching Emacs

** Query & Agenda (=org query=)

- *Query language* with predicates: =todo=, =done=, =tags=, =heading=,
  =property=, =priority=, =level=, =scheduled=, =deadline=, =closed=, =clocked=
- *Boolean operators*: =and=, =or=, =not=, parentheses, implicit AND
- *Agenda view*: scheduled/deadline items grouped by day
- *Tag inheritance*: child entries inherit parent tags in queries
- *Output formats*: human, JSON (with [[file:schemas/org-query-output.schema.json][JSON schema]]), locator (for piping)

** Org Item Locator

A unique identifier for any heading across a tree of org files:

| Pattern                | Example                    | Stability         |
|------------------------+----------------------------+-------------------|
| =id:<uuid>=            | =id:550e8400-e29b-...=     | Survives moves    |
| =<file>::#<custom-id>= | =notes.org::#project-x=    | File-scoped       |
| =<file>::*/<path>=     | =todo.org::*/Work/Meeting= | Fragile on rename |
| =<file>::<line>=       | =inbox.org::42=            | Fragile on edit   |

When an entry has an =:ID:= property, that is always preferred.

** Clock Tracking (=org clock=)

- *Reports*: total clocked time grouped by entry, tag, or day
- *Date filtering*: =--from= and =--to= date range
- *Running clocks*: detect entries with open clock lines
- *Output*: human-readable tables and JSON

** Entry Updates (=org update=)

- *add-id*: add =:ID:= properties to entries (UUID, template, or external command)
- *set-state*: change TODO keyword with automatic =CLOSED: = timestamps
- *add-todo*: insert new entries with planning, priority, and tags
- *add-cookie*: compute and update =[/]= and =[%]= statistic cookies

** Archive (=org archive=)

- Move done entries to an archive file with metadata properties
- Configurable target via =#+ARCHIVE:= or =--target=
- Preserves subtree structure and adds =:ARCHIVE_TIME:=, =:ARCHIVE_FILE:=, =:ARCHIVE_OLPATH:=

** Calendar Export (=org export=)

- *iCalendar* (RFC 5545): SCHEDULED → VEVENT, DEADLINE → VTODO
- *JSCalendar* (RFC 8984): structured JSON with Event/Task types
- Date range and tag filtering, file or stdout output

* Quick Start

#+begin_src sh
# Install
cargo install --path crates/org

# Lint
org fmt check file.org

# Format in place
org fmt format file.org

# Query entries
org query search 'todo:TODO tags:work' ~/org/

# Agenda view (next 7 days)
org query agenda ~/org/

# JSON output for tooling
org query search 'deadline:past' --format json ~/org/

# Clock report grouped by tag
org clock report ~/org/ --group-by tag

# Export scheduled items to iCalendar
org export ical ~/org/ -o calendar.ics

# Add IDs to all entries
org update add-id ~/org/

# Mark entries as DONE via piping
org query search 'todo:TODO tags:done' --format locator | org update set-state DONE --stdin

# Archive completed entries
org archive ~/org/

# Shell completions
org --completions bash > /etc/bash_completion.d/org
#+end_src

* Configuration

Copy =.org-tools.toml.example= to =.org-tools.toml= in your project root:

#+begin_src toml
[format]
trailing_whitespace = true
blank_lines = false
heading_blank_lines = false
table_format = true
property_drawer_align = true

[lint]
disabled_rules = []
#+end_src

org-tools searches for =.org-tools.toml= starting from the current directory and
walking up to the filesystem root.

See [[file:docs/user/usage.org][docs/user/usage.org]] for the full usage guide and rule reference.

* Building from Source

Requires Rust 1.70+ or use [[https://mise.jdx.dev/][mise]] to install the pinned version:

#+begin_src sh
mise install       # install pinned Rust
mise run build     # cargo build
mise run test      # cargo test
mise run lint      # clippy + rustfmt check
#+end_src

* Project Structure

org-tools is a Cargo workspace with two crates:

| Crate            | Purpose                                                                                         |
|------------------+-------------------------------------------------------------------------------------------------|
| =org-tools-core= | Shared library: rules, runner, config, OrgDocument, OrgLocator                                  |
| =org=            | Umbrella binary: =org fmt=, =org query=, =org clock=, =org update=, =org export=, =org archive= |
